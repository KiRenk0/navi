本文件是后续实现 Faceted3D 的唯一依据，如与其它 specs/注释冲突，以本文件为准。
### 目标（给后续 AI 的“继承说明”）
在不破坏原有 2D 工程近似求解器的前提下，新增一个**“Faceted 强三维分支”**：用**分块平面（Faceted）几何**构造局部切平面法向，令迎风面外缘链路从仅依赖 \(dz/dx\) 升级为依赖 \((dz/dx, dz/dy)\) 的 **3D 入流角**，从而在“类 HTV‑2 大后掠扁平升力体”缺少多截面数据时仍能捕捉关键三维压缩效应。

同时支持“自动分流”：**2D 用原 solver，3D 用新 solver**（按 vehicle spec 是否包含 `faceted3d` 字段判断）。

---

### 起因 / 背景（为什么必须做“强 3D 分支”）
- **原工程代码的适用对象**：`specs/airfoils/doublewedge_t0p034_interp_linear.dat` 这类 **2D 截面恒定** 翼型 + 梯形/三角平面形（本质是“2.5D 条带法”）。迎风面外缘链路里核心角度近似为：
  
  \[
  \phi \approx \alpha_e - \arctan(dz/dx)
  \]
  
  它**只看弦向斜率**，展向几何（\(dz/dy\)）不会进入 `phi`。
- **要换的新对象（类 HTV-2 升力体）**：强三维、侧向压缩显著、展向坡度很大；如果仍只用 \(dz/dx\)，等价把它当“无限翼展二维楔形”，关键物理机制丢失。
- **论文信息限制**：原文没有提供多展向截面点集/参数表，只能确定“中轴线（脊线）”与“平面轮廓（前缘线）”。因此选择工程折中：**Faceted（分块平面）假设**。

### 经过 / 讨论结论（本项目采取什么折中）
- **用户偏好约束**：不直接污染原 2D 求解器；复制出一个增强版求解器，并由程序判断走哪个分支。
- **可行性结论**：不需要推倒重写 CFD，只需把“迎风面入流角”的定义从二维升级为三维切平面法向。
- **强三维的最小可行输入**：在 Faceted 假设下，每个面片 \(dz/dx\) 与 \(dz/dy\) 是常数（或分段常数），因此无需点云也能生成斜率场。

### 总体架构设计（后续 AI 按这个实现）
#### 1) 新增 3D 求解器文件（核心）
- 新增文件：`src/ref_enthalpy_method/solver_faceted3d.py`
- 类名建议：`WingLowFidelitySolverFaceted3D`
- 设计原则：
  - **复制** `solver.py` 的“汤底”（气体模型、大气、热流模块、输出结构等尽量复用）。
  - 只替换迎风面外缘链路里“phi 的定义”和“几何输入”的部分。
  - 保持 `compute_snapshot()` 输出字段一致（`q_w/q_l/Tw_w/Tw_l/w_tr/re_edge/re_tri`），这样 `scripts/run_case_rem.py` 后续保存/画图逻辑无需改。

#### 2) 3D 几何模块（提供 \(s_x,s_y\) 场 + planform mask）
- 新增模块建议：
  - `src/ref_enthalpy_method/geometry/faceted3d.py`
- 接口建议（最小可用）：
  - 输入：`vehicle_spec.vehicle_spec.faceted3d` 配置（见下文 spec 设计）
  - 输出：
    - `sx_up, sy_up, sx_lo, sy_lo`：形状 \((ny,nx)\) 或 \((nx,)\)（若面片常数，可广播）
    - `mask_inside`：形状 \((ny,nx)\) 的布尔掩码，表示点是否在三角平面形内（用户已选择“triangle_mask”）
- Faceted 方案（不依赖点云）：
  - 使用脊线（y=0）上/下表面“弦向斜率角”（例如 10°/3°）定义 \(s_x\)（可能是常数或分段常数）。
  - 用 planform 半顶角（18°）定义前缘线：\(y_{LE}(x)=x\tan(\theta)\)（具体坐标系需统一：见“坐标系约定”）。
  - 用“脊线到前缘线高度线性衰减”为平面假设，导出 \(s_y\) 为常数（或随 x 分段）。
  - 若需要上/下表面不同的侧向斜率，用各自的“脊线半模角/帐篷角”推导。

> **关键点**：真正的 3D 效应来自 \(s_y=\partial z/\partial y\)（或等价法向变化）。即使 \(s_x\) 常数也可以产生 3D 影响，只要 phi 的定义使用法向。

#### 3) 迎风面外缘 cache 的 3D 版本（把 phi 从 2D 改为 3D）
现有 2D 逻辑在：
- `src/ref_enthalpy_method/aero/windward_cache.py`
  - `phi = alpha_e - atan(slope_arr[i])`
  - `cp = busemann_cp(ma_inf=mach_eff, phi_rad=phi)`
  - 由 `compute_edge_conditions()` 得到 `EdgeConditions`

3D 版建议新增文件：
- `src/ref_enthalpy_method/aero/windward_cache_faceted3d.py`

新增/替换内容：
- 输入不再是 `slope_arr(x)`，而是每条带：
  - `sx_arr(x)`、`sy_arr(x)`（长度 nx）
- 构造局部法向（未归一）：
  - \(n = (-s_x,\ -s_y,\ 1)\)
- 构造来流方向（在机体系，约定见下）：
  - \(u = (\cos\alpha_e,\ 0,\ -\sin\alpha_e)\)（先不引入侧滑角；后掠影响继续用现有 `effective_alpha`/`effective_ma_inf`）
- 定义 3D 入流角（建议取“与法向的夹角的补角/或直接用法向投影”中稳定的一种），例如：
  - \(\phi = \arcsin\left(\frac{u\cdot n}{\|u\|\|n\|}\right)\)
  - 并沿用已有 phi clamp（`lf_cfg.phi_clamp`）
- 然后用相同的 `busemann_cp()` → `compute_edge_conditions()` 得到 edge。

> 后续 AI 注意：2D 里 phi 的符号约定很敏感；要加单元测试验证“迎风面应为压缩（phi>0）”。

#### 4) Planform mask（用户已选 triangle_mask）
- 在 3D solver 里，针对 2D 采样网格 \((x/c,y/b)\)：
  - 先转换到物理坐标（或用无量纲判断）决定点是否在平面形内：
    - 三角平面形示例：给定半顶角 \(\theta\)，在无量纲下可用 \(y/b \le k \cdot x/c\)（k 的推导取决于你把 “b_half” 和 “c_root” 的定义如何映射到同一坐标系）。
- 对 mask 外的点：
  - `q_w/q_l/Tw_w/Tw_l/w_tr/re_edge/re_tri` 置为 `NaN` 或 0（建议 NaN，更便于后处理识别无效区域）。

### 坐标系与角度约定（必须写进代码注释，避免口径崩）
建议统一如下（后续 AI 若改动要同步所有公式）：
- 机体坐标：
  - \(x\)：沿来流/机体纵向（从头到尾）
  - \(y\)：展向（从中心线到右半边）
  - \(z\)：上为正
- 上表面/下表面高度场：\(z(x,y)\)
- 坡度：
  - \(s_x=\partial z/\partial x\)
  - \(s_y=\partial z/\partial y\)
- 法向（未归一）：\(n=(-s_x,-s_y,1)\)
- 来流方向（仅考虑有效攻角）：\(u=(\cos\alpha_e,0,-\sin\alpha_e)\)
- 3D 入流角 \(\phi\)：用 `asin((u·n)/(|u||n|))`（或另一种清晰定义，但要一致）

> 重要：后掠角 `sweep_le_deg` 在现有模型里用于 `effective_alpha/effective_ma_inf`，这本质是“独立原理”近似。Faceted 3D 分支是否仍用它，需要在代码注释里明确：**我们仍沿用独立原理做 sweep 修正，同时再引入 sy 的三维切平面修正**。

填参规则（必须写清）：
- `sx_*`、`sy_*` 是 **无量纲坡度**（即 \(dz/dx\)、\(dz/dy\)），不是角度；角度必须通过 `tan()` 转换后再填。
- 正负号约定必须与“坐标系与角度约定”一致：\(z\) 向上为正，\(y\) 从中心线指向右半边为正。否则法向与 `phi` 的符号会翻转。
